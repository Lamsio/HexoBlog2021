ES介绍

ES，全名为ECMAScript，是脚本语言的规范，而平时经常编写的JavaScript就是ECMAScript的一种实现，所以ES新特性其实指的就是JavaScript特性。

新特性？

  1. 语法简洁，功能丰富
  2. 框架开发应用

前置知识

  1. JavaScript 基本语法
  2. AJAX 与 NodeJS

ES6 新特性

ES6主要有以下几个新特性:

  1. let关键字

let 关键字

在ES5中，我们只能用var定义变量，但在ES6中则多了两种定义变量的关键字，其中一个就是let

let关键字有以下特点:

  1. 不可重复声明
  1. 块级作用域
  1. 不存在变量提升

const 关键字

拥有let的特性，此外，声明const变量时必须声明初始值（let可以不需要初始值），const变量一旦声明，其值就不能被修改，这点类似于JAVA中的final，但如果对于常量数组/对象的元素进行修改，不会报错。

  • 潜规则: 常量命名都得完全大写

扩展运算符

所谓扩展运算符就是...，扩展运算符能够将数组转换为以,分割的参数序列。

Symbol

Symbol是ES6引入的新的数据类型，他是JavaScript语言的第七种数据类型，是一种类似于字符串的数据类型。

Symbol的特点:

  1. Symbol值是唯一的，用来解决命名冲突的问题
  2. Symbol值不能与其他数据进行运算
  3. Symbol定义的属性对象不能使用for...in进行循环遍历，但是可以通过Reflect.ownKeys来获取对象的所有键名
使用场景

对未知的对象添加自定义方法，倘若我现在想为tommy对象添加sleep方法，但我不是tommy对象的创建者，我压根不知道tommy对象里有什么方法，我担心我写的sleep可能把他原本的sleep覆盖了，这时，我们用Symbol就能很方便地解决这个问题！

迭代器

迭代器（iterator）是一种接口，可以为各种不同的数据结构提供统一的访问机制。任何数据结构只要部署了iterator接口，就能实现遍历操作。

  1. ES6创造了一种新的遍历命令for...of循环，Iterator接口主要供for...of使用
  2. 源生具备iterator接口的数据（可用for...of遍历）
    1. Array
    2. Arguments
    3. Set
    4. Map
    5. String
    6. TypedArray
    7. NodeList
  3. 工作原理
    1. 创建一个指针对象，指向当前数据结构的起始位置
    2. 第一次调用对象的next方法，指针自动指向数据结构的第一个成员
    3. 接下里不断调用next方法，指针会一直往后移动，直到指向最后一个成员。

Set

ES6提供的新的数据结构 Set (集合)。它类似于数组，但成员值是唯一的，集合也实现了iterator接口，所以可以使用扩展运算符以及for...of遍历。

常见方法:

  1. size — 返回集合的元素个数
  2. add — 增加一个新元素，返回当前集合
  3. delete — 删除元素，返回boolean值
  4. has — 判断是否存在某个值，返回boolean值

Map

Es6提供了Map数据结构，他类似于对象，也是键值对的集合，但是"键"的范围不限于字符串，各种类型的值（包括对象）对能当做键。Map也实现了iterator接口，因此可以使用扩展运算符以及for...of遍历。

常见方法:

  1. size — 返回Map的元素个数
  2. set — 增加一个新元素，返回当前Map
  3. get — 根据键返回值
  4. has — 检测Map中是否包含某个元素，返回boolean值
  5. clear — 清空集合，返回undefined

Class类

ES6提供了更接近传统语法的写法，引入了Class的概念，作为对象的模板。通过class关键字，可以定义类，基本上，在ES6中class可以看做只是一个语法糖，他的绝大多数功能，ES5也能做到，新的class写法只是让对象原型的写法更加清晰、更像面向对象编程的语法而已。

知识点:

  1. class 声明类
  2. constructor 定义构造函数初始化
  3. extends 继承父类
  4. super 调用父级构造方法
  5. static 定义静态方法和属性
  6. 父类方法可以被重写

模块化

模块化是指讲一个大的程序文件，拆分成许多小的文件（功能块），然后将小文件在组合到一块。

模块化的优势有以下几点:

  1. 防止命名冲突
  2. 代码复用
  3. 高维护性

ES6之前的模块化规范有:

  1. CommonJS => Nodejs, Browserify
  2. AMD => RequireJS
  3. CMD => SeaJS

模块功能主要有两个命令构成: export 和 import

  • export 命令用于规定模块的对外接口
  • import 命令用于导入其他模块的功能
